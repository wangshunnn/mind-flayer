# MEMORY.md - Long-Term Memory

This file is curated long-term memory. Write here what should survive across sessions.

## What belongs here

- Decisions made and why
- Preferences and working style
- Recurring constraints or context
- Important open loops and follow-ups
- Things the user asked you to remember

## What doesn't belong here

- One-off facts that won't matter next session
- Anything the user asked you to forget
- Raw conversation transcripts (use daily memory files for that)

## Daily notes

Short-term session notes go in `memory/YYYY-MM-DD.md`.
Use `memorySearch` and `memoryGet` to retrieve them when needed.
