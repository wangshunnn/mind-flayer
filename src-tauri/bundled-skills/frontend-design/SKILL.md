# Frontend Design

This skill guides the creation of distinctive, production-grade frontend interfaces. Implement real working code with exceptional attention to aesthetic detail.

## Design Thinking

Before coding, understand the context and commit to a clear aesthetic direction:

- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick a clear direction — brutally minimal, maximalist, retro-futuristic, editorial, brutalist, soft/pastel, luxury/refined, playful, etc.
- **Constraints**: Framework, accessibility requirements, performance targets.
- **Differentiation**: What makes this memorable? Execute one strong concept with precision.

## Implementation

Write working code (HTML/CSS/JS or React/Vue/Svelte) that is:
- Functional and production-ready
- Visually distinctive and cohesive
- Meticulously refined in every detail

## Aesthetics Guidelines

**Typography**
- Choose fonts that are beautiful and interesting — avoid generic defaults like Inter, Roboto, Arial.
- Pair a distinctive display font with a refined body font.
- Use Google Fonts or system-safe alternatives as needed.

**Color & Theme**
- Commit to a cohesive palette. Use CSS variables for consistency.
- Dominant colors with sharp accents outperform timid, evenly-distributed palettes.
- Vary between light/dark themes — don't always default to dark.

**Motion**
- Prioritize CSS-only animations for HTML artifacts.
- Focus on high-impact moments: staggered reveals on load, hover states that surprise.
- Use `transition` and `animation` thoughtfully — not just on everything.

**Layout & Composition**
- Unexpected layouts: asymmetry, overlap, diagonal flow, grid-breaking elements.
- Generous negative space OR controlled density — commit to one.
- Avoid predictable 3-column card grids unless they serve the content well.

**Visual Details**
- Create atmosphere: gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, grain overlays.
- Match visual complexity to aesthetic: maximalist designs need elaborate code; minimalist designs need restraint and precision.

## What to Avoid

- Overused font families: Inter, Roboto, Arial, system-ui for display text
- Clichéd purple-gradient-on-white AI aesthetics
- Generic card layouts, predictable component patterns
- Cookie-cutter designs that lack context-specific character
- Identical aesthetics across different generations — every design should feel unique

## Output

Deliver complete, self-contained code. For HTML artifacts, include all styles inline or in `<style>` tags. For React components, include all necessary imports and CSS-in-JS or Tailwind classes.

Interpret the user's request creatively. Make bold, intentional choices. Bold maximalism and refined minimalism both work — the key is execution.
